# M13 Awards Registry Expansion — Design

Date: 2026-03-07
Prerequisite: `docs/plans/2026-03-07-m13-feasibility-study.md`

## Scope

Add 4 Wikidata-viable candidates to the awards registry:
1. **Peabody Awards** — single-award ceremony, 446 P166 claims
2. **Gotham Awards** — 4 curated categories (2 documentary, 2 general indie)
3. **Emmy documentary categories** — 5-7 new categories under existing `emmys` ceremony
4. **Guggenheim Fellowship** — qualifier-aware SPARQL query for film/video fellows

## Design Decisions

- **AwardDomain for Peabody:** `"documentary"` — pragmatic fit; Peabody surfaces alongside Academy Best Documentary and Spirit Best Documentary in domain queries, which matches our doc-focused use case.
- **AwardDomain for Guggenheim:** New `"fellowship"` domain value. Semantically distinct from craft-based domains.
- **Guggenheim qualifier filtering:** Category-level metadata on `AwardCategory` rather than hardcoded QID detection. Generic and reusable.
- **Gotham category scope:** 4 categories (Best Documentary, Breakthrough Nonfiction Series, Best Feature, Breakthrough Director). Remaining ~10 categories deferred — can add later without code changes.
- **Peabody single-QID pattern:** Q838121 serves as both ceremony entity and award category. No sub-categories. Same QID in `CEREMONIES` and `AWARD_CATEGORIES`. New pattern but no code changes needed.

## Section 1: Registry Data Changes

### AwardDomain

Add `"fellowship"` to the union type:

```typescript
export type AwardDomain =
  | "picture" | "director" | "screenplay" | "cinematography"
  | "editing" | "acting" | "supporting-acting" | "score"
  | "documentary" | "international" | "animated" | "short"
  | "fellowship";
```

### AwardCategory interface

Add optional `qualifier` field:

```typescript
export interface AwardCategory {
  id: string;
  ceremony: string;
  label: string;
  wikidataId: string;
  domain: AwardDomain;
  qualifier?: { property: string; values: string[] };
}
```

When present, `getAwardHistory()` appends a qualifier filter clause to the SPARQL query. Existing categories have no qualifier and are unaffected.

## Section 2: SPARQL Query Changes

### `wikidata-client.ts` — `getAwardHistory()`

Add optional `qualifier` parameter:

```typescript
async getAwardHistory(
  categoryQid: string,
  qualifier?: { property: string; values: string[] }
): Promise<AwardHistoryEntry[]>
```

When `qualifier` is provided, append to the SPARQL WHERE clause:

```sparql
?stmt pq:{property} ?field .
FILTER(?field IN (wd:{values[0]}, wd:{values[1]}, ...))
```

The Guggenheim query becomes:

```sparql
SELECT ?recipient ?recipientLabel ?date ?forWork ?forWorkLabel WHERE {
  ?recipient p:P166 ?stmt .
  ?stmt ps:P166 wd:Q1316544 .
  ?stmt pq:P101 ?field .
  FILTER(?field IN (wd:Q11424, wd:Q34508))
  OPTIONAL { ?stmt pq:P585 ?date }
  OPTIONAL { ?stmt pq:P1686 ?forWork }
  SERVICE wikibase:label { bd:serviceParam wikibase:language "en" . }
}
ORDER BY DESC(?date)
```

### `awards.ts` — `handleGetAwardHistory()`

Pass qualifier from the category to the client:

```typescript
const cat = findCategory(category);
if (!cat) throw new Error(`Unknown award category: ${category}`);
const entries = await wikidataClient.getAwardHistory(cat.wikidataId, cat.qualifier);
```

### No changes needed elsewhere

- `queryAwards()` / `getPersonWins()` / `getPersonNominations()` — already work with any QID in `REGISTERED_QIDS`. Once Q1316544 is registered, Guggenheim fellows' P166 claims surface automatically in `get_person_awards`.
- `getFilmAwards()` — Peabody recipients include TV programs and films. If a film has P166 = Q838121, it will appear in `get_film_awards` results.

## Section 3: QID Verification

Every QID must be verified via SPARQL before registry entry. Verification = confirm the QID returns at least 1 P166 result.

**Already verified (feasibility study):**
- Peabody Q838121 — 446 P166 claims
- Gotham Q1538791 — ceremony entity confirmed
- Gotham Best Documentary Q20978457 — confirmed
- Gotham Breakthrough Nonfiction Series Q109259295 — confirmed
- Guggenheim Q1316544 — 19,570 P166 claims (217 film-qualified)
- Emmy: Q24895159, Q24895051, Q24900788, Q25345783, Q30632982

**Verify during implementation:**
- Gotham Best Feature — search for QID, verify P166 claims
- Gotham Breakthrough Director — search for QID, verify P166 claims
- 2 remaining Emmy doc categories — identify and verify QIDs

If any unverified QID returns 0 P166 claims, skip it. The registry policy is strict: no unverified QIDs.
